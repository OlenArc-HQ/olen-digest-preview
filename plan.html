Residents · public, no login

You (Lorenzo) · internal back-office
Your private tool that generates the weekly post + flyers. Nobody else uses it. It's the engine behind the funnel — stays separate from the customer side.
✓ BUILT (Plan 1 + 2)

Prospects & paying clients · login
On olenarc.com: free signup → see Impact + the For-Your-Org demo (the hook), surrounded by your case studies. Upgrade → the paid customization workspace. This is REVENUE.

Build plan

0
Static demos · NOW
The preview site (Impact + For-Your-Org) = your pitch material. Send the link to a prospect today — no backend needed.
Status: ✓ done. Use it to validate demand before building the portal.
1
Customer Portal MVP · login + free tier ← START HERE
olenarc.com gets “Sign in / Get started” → login → a dashboard: the free Impact view + the For-Your-Org demo, with a “contact us to get your own.” Impact shows demo data first (no data dependency).
Tech: Next.js · Clerk (reuse olenhub's login) · host on Vercel. Outcome: a prospect signs up free, logs in, sees value → leads.
Effort: ~2–4 weeks focused.
2
Paid customization workspace
Per-client workspaces (their own data): grants pipeline, MOU tracker, member inbox, AI assistant, comms. Plus billing + data import (email/Sheets/PDF → AI aggregation).
Tech: + database (Postgres/Supabase) · Stripe (subscriptions) · AI for the assistant. This is the real product — build after Phase 1 proves demand. Effort: months.
3
Real data + one account
Wire Dub + Umami → live Impact numbers; tie olenhub + console + portal into one OlenArc account.
Gated on: the engineer + the app being live (Clerk sharing, the API, analytics). Until then the portal shows DEMO data.

Recommended next: Phase 1

Turn the static demo into a real “sign up + log in” portal on olenarc.com — same content we already built, now behind a free login. Lowest risk, funnels prospects, no dependency on the engineer.
